# Unity 6 미연시 기능 목록

이 문서는 위 장면 단위 기획서를 Unity 6에서 실제 게임으로 만들 때 필요한 기능을 초보자 기준으로 작게 나눈 개발 목록입니다.

각 기능마다 테스트 성공 기준을 함께 적었습니다.

| 순서 | 기능 | 만드는 내용 | 테스트 성공 기준 |
|---:|---|---|---|
| 1 | 기본 화면 만들기 | `Canvas`, 대사창, 이름창, 배경 이미지 영역, 캐릭터 이미지 영역을 만든다. | 게임 실행 시 검은 화면이 아니라 대사창과 UI가 보인다. |
| 2 | 대사 데이터 만들기 | 장면 번호, 화자 이름, 대사 내용, 배경 이름, 캐릭터 이름을 담는 데이터 구조를 만든다. | 테스트용 대사 3개를 코드나 JSON에 넣을 수 있다. |
| 3 | 대사 출력 | 대사창에 현재 대사 내용을 표시한다. | 게임 시작 시 첫 번째 대사가 화면에 나온다. |
| 4 | 클릭으로 다음 대사 넘기기 | 마우스 클릭이나 화면 터치로 다음 대사로 넘어가게 만든다. | 클릭할 때마다 1번 대사에서 2번 대사, 3번 대사 순서로 바뀐다. |
| 5 | 이름 표시 | 현재 말하는 사람 이름을 이름창에 표시한다. | 이네스 대사에는 `이네스`, 카셀 대사에는 `카셀`이 표시된다. |
| 6 | 대사 타이핑 효과 | 글자가 한 번에 나오지 않고 한 글자씩 나오게 한다. | 대사가 천천히 출력되고, 클릭하면 전체 문장이 바로 표시된다. |
| 7 | 배경 이미지 변경 | 장면 데이터에 적힌 배경 이름에 따라 배경 이미지를 바꾼다. | 침실 장면에서는 `bg_brown_bedroom`, 연회장 장면에서는 `bg_palace_ballroom`이 보인다. |
| 8 | 캐릭터 초상화 변경 | 장면 데이터에 적힌 캐릭터 이미지로 초상화를 바꾼다. | 이네스 대사에는 이네스 이미지, 릴리 대사에는 릴리 이미지가 보인다. |
| 9 | 캐릭터 숨기기 | 캐릭터가 없는 나레이션 장면에서는 초상화를 숨긴다. | 나레이션 장면에서 빈 초상화가 남지 않는다. |
| 10 | 선택지 버튼 | 선택지가 있는 장면에서 버튼 2~3개를 보여준다. | 선택지 없는 장면에서는 버튼이 안 보이고, 선택지 있는 장면에서만 버튼이 보인다. |
| 11 | 선택에 따른 분기 | 선택한 버튼에 따라 다른 장면 번호로 이동하게 만든다. | `차갑게 거절한다`를 누르면 009번 장면, `이유를 묻는다`를 누르면 010번 장면으로 간다. |
| 12 | 호감도 저장 | 선택지에 따라 카셀, 세자르, 타국 왕자 등의 호감도 값을 올리거나 내린다. | 카셀 선택지를 고르면 `카셀 호감도 +1`이 실제 변수에 저장된다. |
| 13 | 호감도 확인 UI | 테스트용으로 현재 호감도를 화면에 작게 표시한다. 나중에는 숨겨도 된다. | 선택지를 누를 때마다 호감도 숫자가 바뀌는 것을 볼 수 있다. |
| 14 | 장면 이동 시스템 | 현재 장면 번호를 기준으로 원하는 장면 데이터를 찾아 보여준다. | 선택지가 없어도 다음 번호로 이동하고, 선택지가 있으면 지정된 번호로 이동한다. |
| 15 | 챕터 이동 | 특정 장면이 끝나면 다음 챕터 첫 장면으로 넘어가게 한다. | Chapter 01 마지막 장면 후 Chapter 02 첫 장면으로 이동한다. |
| 16 | 엔딩 분기 | 마지막에 호감도와 선택 기록을 보고 엔딩을 고른다. | 카셀 호감도가 높으면 카셀 엔딩, 독립 점수가 높으면 독립 엔딩으로 간다. |
| 17 | 엔딩 화면 | 엔딩 이름과 짧은 설명을 보여주는 화면을 만든다. | `카셀 해피엔딩`, `독립 엔딩`, `배드엔딩` 같은 결과가 화면에 표시된다. |
| 18 | 세이브 데이터 구조 | 현재 장면 번호, 호감도, 선택 기록을 저장할 구조를 만든다. | 저장할 데이터에 `현재 장면 번호`, `카셀 호감도`, `세자르 호감도`가 들어 있다. |
| 19 | 세이브 | 버튼을 누르면 현재 진행 상황을 파일로 저장한다. | 게임을 껐다 켜도 저장 파일이 남아 있다. |
| 20 | 로드 | 저장 파일을 읽어서 이전 장면과 호감도를 복원한다. | 030번 장면에서 저장 후 다시 불러오면 030번 장면부터 시작한다. |
| 21 | 타이틀 화면 | `처음부터`, `불러오기`, `종료` 버튼을 만든다. | 처음부터를 누르면 001번 장면, 불러오기를 누르면 저장된 장면으로 간다. |
| 22 | 설정 화면 | 텍스트 속도, 배경음 크기, 효과음 크기를 조절한다. | 텍스트 속도를 바꾸면 글자 출력 속도가 달라진다. |
| 23 | 전체 테스트 | 처음부터 엔딩까지 한 번 플레이해 본다. | 중간에 멈추지 않고 최소 1개 엔딩까지 도착한다. |

## 가장 먼저 만들 핵심 5개

| 우선순위 | 기능 |
|---:|---|
| 1 | 대사 출력 |
| 2 | 클릭으로 다음 대사 넘기기 |
| 3 | 이름 표시 |
| 4 | 배경 이미지 변경 |
| 5 | 선택지 버튼 |

이 5개만 완성해도 미연시의 기본 뼈대가 만들어집니다.

그다음 `호감도 저장`, `엔딩 분기`, `세이브/로드`를 붙이면 게임다운 구조가 됩니다.

## 초보자용 개발 순서 요약

1. 화면 UI를 먼저 만든다.
2. 대사 데이터를 만든다.
3. 대사를 화면에 출력한다.
4. 클릭하면 다음 대사로 넘어가게 한다.
5. 이름, 배경, 캐릭터 이미지를 장면에 맞게 바꾼다.
6. 선택지 버튼을 만든다.
7. 선택지에 따라 다른 장면으로 이동하게 한다.
8. 선택에 따라 호감도를 저장한다.
9. 마지막에 호감도로 엔딩을 고른다.
10. 세이브와 로드를 붙인다.
